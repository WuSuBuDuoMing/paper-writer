# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Paper Writer, please report it responsibly:

1. **DO NOT** open a public GitHub issue for security vulnerabilities
2. Use [GitHub's private vulnerability reporting](https://github.com/WuSuBuDuoMing/paper-writer/security/advisories/new) or email the maintainer at WuSuBuDuoMing@users.noreply.github.com
3. Include as much detail as possible:
   - Description of the vulnerability
   - Steps to reproduce the issue
   - Potential impact assessment
   - Suggested fix (if any)

We will acknowledge receipt within 48 hours and provide an estimated timeline for a fix.

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| Latest  | Yes                |
| Older   | No (please upgrade)|

We recommend always using the latest release to benefit from security fixes and improvements.

## Response Time

| Severity | Response Time |
|----------|--------------|
| Critical | 24-48 hours  |
| High     | 3-5 days     |
| Medium   | 7 days       |
| Low      | 14 days      |

## Security Best Practices

When using Paper Writer in your projects:

- Keep your Claude Code installation up to date
- Review agent definitions before copying them to your projects
- Do not commit sensitive research data alongside agent configurations
- Use `.gitignore` to exclude any locally generated paper drafts containing unpublished data

## Disclosure Policy

- We will confirm receipt of your vulnerability report within 48 hours
- We will provide an estimated timeline for a fix
- We will notify you when the vulnerability has been fixed
- We will publicly disclose the vulnerability after a fix has been released

Thank you for helping keep Paper Writer and its community safe!
